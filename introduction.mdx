Hiszpan Scripts is a collection of ready-to-use scripts and a lightweight command-line interface designed to eliminate the repetitive work that slows developers down. Rather than rewriting the same build, lint, test, and deployment steps across every project, you get a consistent, composable toolkit you can drop into any codebase and use immediately — no complex setup, no vendor lock-in.

## Who it's for

Hiszpan Scripts is built for any developer or team that manages recurring workflow tasks from the command line. It's a good fit if you:

- find yourself copying the same shell scripts between projects,
- want a single, uniform interface for tasks that currently live in scattered `package.json` scripts or Makefiles,
- need to onboard new contributors quickly with a predictable, self-documenting set of commands, or
- want to automate multi-step workflows from a CI pipeline, cron job, or local terminal without a heavyweight orchestration tool.

## Core concepts at a glance

Understanding three ideas covers most of what you'll need day-to-day:

- **Scripts** — named units of work, either built-in or custom, that you invoke with `script run <name>`.
- **Configuration** — a `hiszpan.config.json` file in your project root that controls script behavior, environment options, and default values.
- **Environment variables** — a set of `HISZPAN_*` variables (`HISZPAN_SCRIPTS_DIR`, `HISZPAN_VERBOSE`, `HISZPAN_CONFIG`, `HISZPAN_SHELL`) that let you override settings without editing the config file, which is useful in CI environments.

## Key features

- **Minimal footprint** — the core CLI has a small, audited dependency tree to keep your project lean and your supply chain manageable.
- **First-class TypeScript support** — full type definitions ship with the package so custom scripts get autocomplete and type-checking out of the box.
- **Consistent output** — every script uses the same structured output format, making logs easy to parse in automated pipelines.
- **Global flags everywhere** — `--help`, `--version`, `--verbose`, and `--config <path>` work on every command, not just a subset.
