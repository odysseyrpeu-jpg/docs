---
title: "Write and Run Your First Automation Script"
sidebarTitle: "First Script"
description: "Create a custom automation script from scratch and run it with the Hiszpan Scripts CLI in a few simple steps."
---

In this guide, you'll initialise a Hiszpan Scripts project, write a simple greeting script, and run it from the command line. By the end, you'll understand the core workflow — write a shell script, drop it in your scripts directory, and let the CLI handle the rest. This pattern scales to any automation task you need to repeat across your project.

<Steps>
  <Step title="Initialise your project">
    From your project root, run the `init` command to generate a `hiszpan.config.json` file with sensible defaults:

    ```bash
    script init
    ```

    Expected output:

    ```text
    Created hiszpan.config.json
    ```

    The generated config file looks like this:

    ```json hiszpan.config.json
    {
      "scriptsDir": "scripts",
      "defaultShell": "bash",
      "verbose": false
    }
    ```

    This tells the CLI to discover scripts in the `scripts/` directory relative to your project root.
  </Step>

  <Step title="Create the scripts directory">
    Create the directory that Hiszpan Scripts will watch for automation scripts:

    ```bash
    mkdir scripts
    ```

    Hiszpan Scripts auto-discovers any executable file placed inside the directory specified by `scriptsDir`. No manual registration step is needed — just drop a script in and it becomes available to the CLI.
  </Step>

  <Step title="Create the script file">
    Create a new file called `greet.sh` inside the `scripts/` directory and make it executable:

    ```bash
    touch scripts/greet.sh
    chmod +x scripts/greet.sh
    ```

    Open `scripts/greet.sh` in your editor and add the following:

    ```bash scripts/greet.sh
    #!/usr/bin/env bash
    set -euo pipefail

    NAME="${1:-World}"

    echo "Hello, ${NAME}! Welcome to Hiszpan Scripts."
    ```

    The script reads its first positional argument (`$1`) as the name to greet. If no argument is supplied, it falls back to `"World"`.
  </Step>

  <Step title="Run your script">
    Execute the `greet` script with the CLI:

    ```bash
    script run greet
    ```

    You should see the following output in your terminal:

    ```text
    Hello, World! Welcome to Hiszpan Scripts.
    ```
  </Step>

  <Step title="Pass a parameter to your script">
    Supply a custom name by appending it after the `--` separator. The double dash tells the CLI that everything following it should be forwarded as raw arguments to the script itself, rather than being interpreted as CLI flags.

    Because `greet.sh` reads a positional argument (`$1`), pass the value directly:

    ```bash
    script run greet -- Alice
    ```

    Expected output:

    ```text
    Hello, Alice! Welcome to Hiszpan Scripts.
    ```
  </Step>
</Steps>

<Tip>
  Run `script list` at any time to see every script that Hiszpan Scripts has discovered in your `scriptsDir`. If `greet` doesn't appear in the list, double-check that the file is executable (`chmod +x scripts/greet.sh`) and that `scriptsDir` in your `hiszpan.config.json` points to the correct directory.
</Tip>
