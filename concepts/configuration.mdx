The `hiszpan.config.json` file is the single source of truth for how Hiszpan Scripts behaves in your project. Place it at your project root and commit it to version control so that every contributor and CI environment runs scripts with the same settings. You don't need a config file to get started — Hiszpan Scripts falls back to sensible defaults — but defining one gives you full control over discovery, environment injection, lifecycle hooks, and output.

## Example configuration

```json
{
  // Directory where your scripts live (relative to this config file)
  "scriptsDir": "./scripts",

  // Shell used to execute shell scripts
  "defaultShell": "bash",

  // Environment variables injected into every script run
  "env": {
    "NODE_ENV": "production",
    "API_BASE_URL": "https://api.example.com"
  },

  // Lifecycle hooks that wrap every script execution
  "hooks": {
    "before": "scripts/hooks/before.sh",
    "after": "scripts/hooks/after.sh"
  },

  // Stream extra diagnostic output to the terminal
  "verbose": false
}
```

## Configuration fields

<ParamField body="scriptsDir" type="string" default="./scripts">
  The directory Hiszpan Scripts scans when looking up a script by name. The path is resolved relative to the location of `hiszpan.config.json`. You can point this at any directory in your project — for example, `"./automation"` or `"../../shared/scripts"` in a monorepo.
</ParamField>

<ParamField body="defaultShell" type="string" default="bash">
  The shell binary used to execute shell scripts (`.sh` files). Common values are `"bash"`, `"sh"`, and `"zsh"`. JavaScript scripts (`.js`) always run through the Node.js binary and are not affected by this setting.
</ParamField>

<ParamField body="env" type="object">
  A flat key-value map of environment variables that Hiszpan Scripts injects into the process environment before every script runs. Values defined here are merged with — and take precedence over — the shell's existing environment. Use this field for shared constants such as API endpoints, stage names, or feature flags that multiple scripts need.

  ```json
  "env": {
    "DEPLOY_REGION": "us-east-1",
    "LOG_LEVEL": "info"
  }
  ```
</ParamField>

<ParamField body="hooks" type="object">
  An object with two optional keys, `before` and `after`, each pointing to a script path (relative to the config file). The `before` hook runs immediately before the target script, and the `after` hook runs immediately after — even if the target script exits with a non-zero code. Use hooks for tasks like acquiring locks, sending notifications, or cleaning up temporary files.

  ```json
  "hooks": {
    "before": "scripts/hooks/notify-start.sh",
    "after": "scripts/hooks/notify-end.sh"
  }
  ```
</ParamField>

<ParamField body="verbose" type="boolean" default="false">
  When set to `true`, Hiszpan Scripts prints additional diagnostic information to `stdout` during execution: the resolved script path, injected environment variables, hook invocations, and timing data. This is useful for debugging unexpected behaviour without modifying your scripts.
</ParamField>

## Environment variable overrides

You can override select configuration values at runtime using environment variables, without editing `hiszpan.config.json`. This is especially useful in CI pipelines where you want a single config file but need environment-specific behaviour.

| Environment variable | Overrides config field | Example |
|---|---|---|
| `HISZPAN_SCRIPTS_DIR` | `scriptsDir` | `HISZPAN_SCRIPTS_DIR=./ops/scripts script run deploy` |
| `HISZPAN_VERBOSE` | `verbose` | `HISZPAN_VERBOSE=true script run build` |

Environment variable overrides take the highest precedence — they win over both `hiszpan.config.json` values and built-in defaults.

<Tip>
  Store secrets like API keys or tokens in your CI platform's secret store and inject them via the shell environment rather than hardcoding them in the `env` block of `hiszpan.config.json`. The config file is typically committed to version control, so it should only contain non-sensitive values.
</Tip>

<Tip>
  Run `script config` at any time to print the fully resolved configuration for your current directory, including all environment variable overrides applied. This is the fastest way to confirm what Hiszpan Scripts will actually use when you execute a script.
</Tip>
