This page covers all global CLI flags and environment variables that apply across every Hiszpan Scripts command, as well as the exit codes your scripts and CI pipelines may encounter. Global flags can be combined with any subcommand, and environment variables let you configure behavior without modifying `hiszpan.config.json` — making them ideal for CI environments, Docker containers, and shared machines.

---

## Global flags

These flags are available on every command. When combined with a subcommand's own flags, they must appear before or after the subcommand name.

<ParamField body="--help" type="flag">
  Prints help text for the current command (or top-level usage when no
  subcommand is given) and exits immediately.
</ParamField>

<ParamField body="--version" type="flag">
  Prints the installed version of Hiszpan Scripts and exits immediately.
</ParamField>

<ParamField body="--verbose" type="flag">
  Enables verbose output across all commands. When set, Hiszpan Scripts emits
  resolved paths, shell invocations, environment snapshots, and exit codes to
  stdout. Equivalent to setting `HISZPAN_VERBOSE=1`.
</ParamField>

<ParamField body="--config <path>" type="string">
  Path to a custom configuration file. Overrides the default lookup for
  `hiszpan.config.json` in the current directory. Useful when you maintain
  multiple configs for different environments or monorepo packages. Equivalent
  to setting `HISZPAN_CONFIG=<path>`.
</ParamField>

## Environment variables

You can use the following environment variables to configure Hiszpan Scripts without modifying any files. They are especially useful in CI/CD pipelines and containerized environments where you need ephemeral, per-run configuration.

<Note>
  Environment variables take precedence over values defined in
  `hiszpan.config.json`. If the same option is set in both places, the
  environment variable always wins. This lets you safely override config file
  values at runtime without editing the file itself.
</Note>

| Variable | Type | Description |
|---|---|---|
| `HISZPAN_SCRIPTS_DIR` | `string` (path) | Overrides the `scriptsDir` value in your config file. Set this to an absolute or relative path to point Hiszpan Scripts at a different scripts directory at runtime. |
| `HISZPAN_VERBOSE` | `0` or `1` | Set to `1` to enable verbose output globally, equivalent to passing `--verbose` on every command. Set to `0` (or leave unset) for standard output. |
| `HISZPAN_CONFIG` | `string` (path) | Path to a custom config file, equivalent to `--config <path>`. When set, Hiszpan Scripts skips the default `hiszpan.config.json` lookup entirely. |
| `HISZPAN_SHELL` | `string` (path) | Overrides the default shell used to execute scripts (e.g. `/bin/zsh`, `/usr/bin/fish`). Defaults to `/bin/bash` when not set. |

**Example — setting variables for a single run:**

```bash
HISZPAN_VERBOSE=1 HISZPAN_SCRIPTS_DIR=ci/scripts script run build
```

**Example — exporting variables for a CI pipeline:**

```yaml
# GitHub Actions example
env:
  HISZPAN_CONFIG: .config/hiszpan.ci.json
  HISZPAN_VERBOSE: "1"
  HISZPAN_SHELL: /bin/bash
```

## Exit codes

Hiszpan Scripts uses the following exit codes. You can check them in shell scripts with `$?` or in CI pipeline conditions to distinguish between different failure modes.

| Code | Constant | Meaning |
|---|---|---|
| `0` | `SUCCESS` | The command completed successfully. |
| `1` | `ERR_GENERAL` | A general or unexpected error occurred. Check stderr for details. |
| `2` | `ERR_SCRIPT_NOT_FOUND` | The specified script name was not found in the configured `scriptsDir`. |
| `3` | `ERR_CONFIG` | A configuration error was encountered — for example, a missing or malformed `hiszpan.config.json`. |
| `127` | `ERR_COMMAND_NOT_FOUND` | The shell command or binary referenced inside the script could not be found on `PATH`. |

**Example — checking exit codes in a shell script:**

```bash
script run deploy
EXIT_CODE=$?

if [ $EXIT_CODE -eq 0 ]; then
  echo "Deployment succeeded."
elif [ $EXIT_CODE -eq 2 ]; then
  echo "Error: 'deploy' script not found. Run 'script list' to see available scripts."
elif [ $EXIT_CODE -eq 3 ]; then
  echo "Error: configuration problem. Check your hiszpan.config.json."
else
  echo "Deployment failed with exit code $EXIT_CODE."
fi
```
