This page covers everything you need to install Hiszpan Scripts in your environment. You'll learn about system requirements, how to install locally or globally, and how to verify, update, and uninstall the package.

## Requirements

Before you install, make sure your system meets the following prerequisites:

- **Node.js 18 or higher** — Hiszpan Scripts relies on features introduced in Node.js 18.
- **npm 8+, yarn 1.22+, or pnpm 7+** — any of these package managers will work.

<Note>
  Not sure which version of Node.js you're running? Check with `node --version`. If you need to upgrade, use [nvm](https://github.com/nvm-sh/nvm) or download the latest LTS release from [nodejs.org](https://nodejs.org).
</Note>

## Install

### Local installation (recommended)

Install Hiszpan Scripts as a development dependency within your project. This keeps the version locked to your project and makes it reproducible across all environments.

<CodeGroup>
  ```bash npm
  npm install --save-dev hiszpan-scripts
  ```

  ```bash yarn
  yarn add --dev hiszpan-scripts
  ```

  ```bash pnpm
  pnpm add --save-dev hiszpan-scripts
  ```
</CodeGroup>

### Global installation

Install Hiszpan Scripts globally to use the `script` command anywhere on your machine without a per-project install.

<CodeGroup>
  ```bash npm
  npm install -g hiszpan-scripts
  ```

  ```bash yarn
  yarn global add hiszpan-scripts
  ```

  ```bash pnpm
  pnpm add -g hiszpan-scripts
  ```
</CodeGroup>

<Warning>
  On macOS and Linux, global installations may require elevated permissions. If you see a permissions error, you can prefix the command with `sudo` — for example, `sudo npm install -g hiszpan-scripts`. For a safer long-term solution, consider [configuring npm to install global packages without sudo](https://docs.npmjs.com/resolving-eacces-permissions-errors-when-installing-packages-globally).
</Warning>

## Verify Installation

After installing, confirm that the `script` command is available and pointing to the correct version:

```bash
script --version
```

You should see output like this:

```
hiszpan-scripts/1.0.0 darwin-arm64 node-v20.11.0
```

If the command is not found after a local install, you can invoke it directly through your package manager:

```bash
npx hiszpan-scripts --version
```

## Update

To update Hiszpan Scripts to the latest version, run the install command again with the `@latest` tag:

<CodeGroup>
  ```bash npm
  npm install --save-dev hiszpan-scripts@latest
  ```

  ```bash yarn
  yarn add --dev hiszpan-scripts@latest
  ```

  ```bash pnpm
  pnpm add --save-dev hiszpan-scripts@latest
  ```
</CodeGroup>

For a global install, use the same approach with the `-g` flag:

```bash
npm install -g hiszpan-scripts@latest
```

## Uninstall

To remove Hiszpan Scripts from a project, uninstall it using your package manager:

<CodeGroup>
  ```bash npm
  npm uninstall hiszpan-scripts
  ```

  ```bash yarn
  yarn remove hiszpan-scripts
  ```

  ```bash pnpm
  pnpm remove hiszpan-scripts
  ```
</CodeGroup>

To remove a global installation, run:

```bash
npm uninstall -g hiszpan-scripts
```
